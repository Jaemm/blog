## 4. Pined repo 활용하기

README 아래 pinned 부분에 내 repository들을 핀으로 지정해서 미리보기 할수 있는 공간이 있다 아래와 같이 꾸며보자
![github-profile readme 본문](/static/images/github/github-profile-setting6.png)

- prodcutive-box 커밋시각 통계 노출
  ![github-profile readme 본문](/static/images/github/github-profile-setting7.png)

우선 해당 레포지토리를 fork한다 (https://github.com/techinpark/productive-box)

github gist 에 들어가서 새로운 public gist를 생성한다(https://gist.github.com/)

![github-profile readme 본문](/static/images/github/github-profile-setting8.png)
내용은 아무렇게나 작성해도 된다

중요. create public gist를 클릭, 해당 페이지 url에서 https://gist.github.com/사용자아이디/해당 부분 기억하기

github 보안 토큰 생성
`settings > developer settings > personal access tokens > tokens
repo, user, gist 선택`한 채로 생성
이때 토큰을 꼭 복사해 두어야한다(다시 확인이 불가)

fork한 레포지토리의 .github/workflow/Schedule.yml 파일을 수정하여 환경변수 작성
해당 레포지토리 `settings로 이동 > security > secrets and variables > actions > new repository secret 생성`

![github-profile readme 본문](/static/images/github/github-profile-setting9.png)
GH_TOKEN, GIST_ID 입력

- 내 프로필에서 gist pin 고정
